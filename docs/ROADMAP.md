# MPVault 产品路线图

## 北极星

让用户能把平台内容变成自己长期拥有、随时找得到、可以继续使用的知识资产。

路线图按用户任务组织，不按技术模块堆功能。排序同时考虑用户价值、投票、实现成本、微信接口稳定性和隐私风险。

## 现在：带回来，而且能证明

- 全量 Markdown 与本地图片。
- 原始 ID 精确核验。
- 检查点、断点续传与状态 manifest。
- 空正文、缺失文件和本地图片校验。
- 多关键词离线全文检索。
- Codex Skill 自动执行与结果汇报。

## Capture：更省心地持续收集

| 候选能力 | 用户价值 | 成本 | 隐私风险 |
| --- | --- | --- | --- |
| 增量同步与新增/变更报告 | 不必反复全量检查 | 中 | 低 |
| 定时备份与失败通知 | 真正做到无人值守 | 中 | 中 |
| 多公众号工作区 | 统一管理研究来源 | 中 | 低 |
| 标题/作者/日期/原创/合集筛选 | 只拿需要的内容 | 低 | 低 |

## Preserve：保存得更完整、更省空间

| 候选能力 | 用户价值 | 成本 | 隐私风险 |
| --- | --- | --- | --- |
| HTML 原排版归档 | 保留视觉与复杂互动内容 | 中 | 低 |
| 视频、音频和附件本地化 | 不只保存图文 | 高 | 低 |
| 图片内容哈希去重 | 大型档案显著瘦身 | 中 | 低 |
| 完整性快照与变更检测 | 发现本地文件损坏或被改动 | 中 | 低 |

## Find：从“有文件”到“找得到”

| 候选能力 | 用户价值 | 成本 | 隐私风险 |
| --- | --- | --- | --- |
| 本地阅读与搜索界面 | 非技术用户也能浏览 | 中 | 低 |

| 语义搜索 | 不记得原词也能找到文章 | 中 | 中 |

## Use：让内容进入下一条工作流

| 候选能力 | 用户价值 | 成本 | 隐私风险 |
| --- | --- | --- | --- |
| JSON / Excel / TXT | 分析和迁移更容易 | 低 | 低 |
| Docx / PDF | 分享与打印 | 中 | 低 |
| Obsidian MOC、标签和双链 | 直接融入个人知识库 | 中 | 低 |
| RSS 与只读 API | 接入自动化和其他应用 | 中 | 中 |

## Understand：在安全边界内理解传播

阅读、点赞、转发、评论与回复需要独立的短期 credentials。它们有真实价值，但敏感度和失效速度都高于正文，因此必须满足以下条件才会实现：

- 与正文登录会话分开保存。
- 默认不采集、不打印、不进入 manifest。
- 明确展示失效时间与删除方式。
- 用户主动开启并理解风险。

## 怎么投票

打开 [Roadmap Issues](https://github.com/GongziTANG/mpvault/issues?q=is%3Aissue%20label%3Aroadmap)，在你需要的功能下点 👍。没有对应条目时，用 Feature Request 模板描述你的场景，而不只是写一个功能名。

Star 代表“这个方向值得继续”，Issue reaction 代表“下一步先做这个”。
